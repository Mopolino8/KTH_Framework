Detailed Description

Arnoldi algorithm with PARPACK.

Description of the method can be found in [Peplinski2014] [Peplinski2015]

Module interface:
This module provides following interface to tstepper:
  • stepper_param_in
  • stepper_param_out
  • stepper_vinit
  • stepper_solve
Module parameters:
Global parameter list:
Parameters provided by arnoldi_arpack submodule (include file - ARNOLDI_ARPACKD):
  • ARNKRYLOV : (integer) size of Krylov space
  • ARNEGV : (integer) number of calculated eigenvalues
  • ARNISTART : (integer) checkpoint file number for restart (reading)
  • ARNISTOP : (integer) checkpoint file number for saving data
Global module dependency:
Parameters required by arnoldi_arpack submodule:
  1. CHKPOINTD:
    • IFCHKPTRST : if restart
    • CHKPTSTEP - checkpiont dump frequency (number of time steps)
  2. TIME_STEPPERD:
    • TSTMODE :
      • 0 : no arnoldi
      • 1 : direct mode
      • 2 : adjoint mode
      • 3 : initial optimal condition
    • TSTSTEP : frequency of calling arn_vsolve (number of time steps)
    • TSTCMAX : maximal number of arnoldi cycles
    • TSTTOL : PARPACK tolerance for Ritz values
  3. CONHT (for conjugated heat transfer):
    • CHCST_SC, CHCFF_V, CHCFF_T : velocity and temperature scaling factors
    • CHGRX, CHGRY, CHGRZ : gravitational acceleration
    • CHRA, CHPR : Rayleight and Prandtl numbers
  4. Nek5000 PARAM array
    • PARAM(66) - write format
    • PARAM(67) - read format
Module parameter usage:
&arpack
arnkrylov= 100,
arnegv= 10,
arnistart= 1,
arnistop= 2,
/

Two possible modes are supported:

  • direct: A*x = lambda*x
  • inverse: A*x = lambda*M*x, M-mass matrix defining inner product

Inverse mode is preferred as it takes into account inner product Simulation with temperature or passive scalars has to be performed in inverse mode due to speciffic inner product.

ARPACK_DIRECT preprocessing switch between direct and inverse mode.

Attention
This submodule has its own checkpointing routines.
