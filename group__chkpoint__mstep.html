Detailed Description

Multistep checkpointing for DNS and linear simulations.

Deprecated:
This module is deprecated, but it is here for backward compatibility.

In the case of multistep time-integration method one needs data from NBDINP timestep. I use standard full_restart and full_restart_save subroutines. There are four 'rsX...' reatart files saved in double precission. Only .f format is supported. In any case two sets of restart files (e.g. 1-3;4-6) are created.

Attention
To make this version to work correctly mod(total_number_of_steps,chkp_step).ge.NBDINP Otherwise last checkpoint will not be complete. Moreover Nek5000 will always start from writing the first file set.
Module interface:
Global interface list:
Interface provided to checkpoint:
  1. chkpt_init

Module parameters:
Global parameter dependency:
Parameters required by multistep module:
  1. CHKPOINTD:
    • chpt_step - checkpiont dump frequency (number of time steps)
    • chpt_ifrst - if restart
    • chpt_fnum - restart set number
  2. Nek5000 PARAM array
    • PARAM(66) - write format
    • PARAM(67) - read format
