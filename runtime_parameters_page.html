Runtime parameters for KTH framework.

One of the goals for KTH framework is to provide organised structure for runtime parameters, that would allow for simple modiffication of parameter list without introducing conflicts between different code parts.

I this solution parameters can be defined by modules. Every module introducing new set of parameters should provide paramerer reading/writing routines, that could be called by runprm_in and runprm_out respectively. Parameters are groupped into sections corresponding to modules and storred in a setup.rpar file using Fortran90 namelist. The runtime parameter file has a followitn structure:

&tstepper
tstmode= 1,
tststep= 40,
tstcmax= 200,
tsttol= 1.0000000000000000e-06 ,
tstifuzawa=t,
/
&arpack
arnkrylov= 100,
arnegv= 10,
arnistart= 1,
arnistop= 2,
/

Each parameter reader should contain following sections:

  • namelist defitionion
  • default parameter values
  • reading from the file preceded by file rewind
  • broadcasting data
